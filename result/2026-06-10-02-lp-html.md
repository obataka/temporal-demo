### ⚠️ SRI（Subresource Integrity）について

Tailwind CSS Play CDN (`https://cdn.tailwindcss.com`) は動的 CDN のため固定の SRI ハッシュを提供していません。本 LP はデモ・マーケティング目的の静的ページであり、ユーザー認証や機密データを扱わないため現状許容としています。

本番ランディングページに昇格する際は、`tailwind` の CLI ビルドで CSS をローカル生成し、CDN 依存を排除することを推奨します。

## 設計の意図・クリティカルポイント

### なぜ 7 セクション構成か
計画では 6 セクションだったが、sticky header を独立した `<header>` 要素として分離することで、スクロール追従ナビと各セクションの責務を明確に分離した。

### クリティカルポイント 3 点

1. **フェーズタイムライン縦線の実装** — `.phase-line::before` 疑似要素で縦線を描画。最終フェーズ（Phase 5）のみ `phase-line` クラスを外して線を出さない設計。Tailwind CDN のみでは任意の疑似要素が使えないため `<style>` ブロックに追記した。

2. **Writer/Reviewer バッジの存在感** — Solution セクションの Phase 4 カード内に、本番 UI と同じ emerald（Writer）/ amber（Reviewer）カラーのバッジを配置。デモを見た商談相手が LP でもリアルなバッジを認識できるよう設計。

3. **フォームはダミー実装** — `<form action="#" method="post">` であり、送信しても何も起きない。ヒアリング枠の実際の受付フローが確定した時点で `action` を更新するだけで接続完了。

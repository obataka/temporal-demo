## 設計の意図・クリティカルポイント

**「3ステップの連動」という差別化ポイントの可視化**  
従来の SOLUTION セクションは「5フェーズの自動化」を順序立てて説明していたが、  
それだけでは「単なるMarkdown化 + レビューツール」と誤読されるリスクがあった。  
TRUE VALUE セクションはフレーミングを変え、「自律監査 → 人間介入 → インフラ連動」の  
3ステップが1つの連動基盤として設計されていること（3ステップどれが欠けても成立しない）を明示した。

**クリティカルポイント 3 点**

1. **LEFT BORDER カラー戦略** — カード1(emerald)・カード2(amber)・カード3(indigo) は  
   SOLUTION セクションの Writer/Reviewer バッジ色と意図的に揃えている。  
   視覚的に「あの議論のこと」とスムーズに結びつく。

2. **Docker/データ流出ゼロの訴求位置** — セキュリティ要件（機密データ流出ゼロ）は  
   「対象企業」ブロックではなく「PoC に含まれるもの」リストに入れた。  
   「含まれるもの」＝提供側の責任として明記することで信頼感が上がる。

3. **構文検証 ExitCode 0 確認済み** — `html.parser` で構文エラーなし。  
   Tailwind CDN 依存のため CSS の動作確認はブラウザで行うこと。

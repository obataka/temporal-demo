## 標準作業手順書（SOP）- Temporal SOP 生成 Activity — フェーズ別 Google Gemini 呼び出しActivity

### 1. はじめに
*   **対象読者:** 全ての関係者（ドキュメントアーキテクト、開発者、運用担当者）
*   **所要時間の目安:** 5分
本SOPは、Temporal Activityとして動作する「SOP生成フェーズ別Google Gemini呼び出しActivity」の目的、構造、および運用方法について記述する。

#### 1.1. ドキュメントの目的とスコープ
本SOPは、Temporal Workflow内で動作するActivityである `generate_sop_phase_activity` の詳細を記述する。このActivityは、標準作業手順書（SOP）を自動生成するプロセスにおいて、特定のフェーズ（章立て提案、詳細執筆、最終レビュー）に応じてGoogle Geminiモデルを呼び出し、その応答を処理することを目的とする。これにより、SOP生成Workflowの各段階でのLLM（大規模言語モデル）とのインタラクションを標準化し、信頼性と保守性を向上させる。

本SOPは、`generate_sop_phase_activity` Activityの設計、実装、使用方法、および運用上の考慮事項に焦点を当てる。

#### 1.2. 対象読者と前提知識
本SOPが想定する読者層（役割）と、彼らが持つべき基本的な技術的知識レベルを説明する。
*   **対象読者:**

    *   Pythonプログラミングの基礎知識。

    *   Markdown形式でのドキュメント作成に関する理解。

#### 1.3. 用語集
SOP内で使用される主要な専門用語（例: Temporal Activity, Workflow, LLM, Google Geminiなど）の定義を提供する。
*   **SOP (Standard Operating Procedure):** 標準作業手順書。特定の作業やプロセスを標準化するための詳細な手順書。
*   **Temporal:** 分散システムの耐障害性とスケーラビリティを向上させるためのオープンソースのオーケストレーションプラットフォーム。
*   **Workflow:** Temporalにおける、一連のビジネスロジックをオーケストレーションするコードの実行単位。長時間実行され、耐障害性を持つ。
*   **Activity:** Temporal Workflowから呼び出される、具体的なタスク（例: 外部API呼び出し、データベース操作）を実行するコードの単位。Activityは冪等であるべきであり、リトライ可能である。
*   **LLM (Large Language Model):** 大規模言語モデル。人間のようなテキスト生成、理解、要約などを行うAIモデル。本SOPではGoogle Geminiモデルを指す。
*   **Google Gemini API:** Googleが提供するLLM「Gemini」にプログラムからアクセスするためのAPI。
*   **System Instructions (システム指示):** LLMに対して、その役割や振る舞いを指示するためのプロンプト。モデルの応答の質と方向性を決定する上で重要である。
*   **Prompt (プロンプト):** LLMに与える入力テキスト。
*   **Task Queue:** Temporal WorkerがActivityタスクを受け取るためのキュー。
*   **Retry Policy:** TemporalがActivityやWorkflowの失敗時に再試行する際のルール（最大試行回数、バックオフ間隔など）を定義するポリシー。

### 2. システム概要
*   **対象読者:** 開発者、ドキュメントアーキテクト
*   **所要時間の目安:** 10分
このActivityは、SOP生成Workflowにおいて、指定されたフェーズ（章立て提案、詳細執筆、最終レビュー）に応じてGoogle Geminiモデルを呼び出す役割を担う。

#### 2.1. `generate_sop_phase_activity`の役割
本ActivityがSOP生成パイプライン全体の中でどのような機能的役割を果たすかを説明する。
`generate_sop_phase_activity` Activityは、SOP生成Workflowの中核を担う機能であり、以下の役割を果たす。

1.  **フェーズに応じたLLMの振る舞い制御:** SOP生成プロセスの各フェーズ（章立て提案、詳細執筆、最終レビュー）に応じて、Google Geminiモデルに異なる「役割」と指示を与える。
2.  **動的なプロンプト生成:** SOP生成に必要な情報（対象のソースコード、承認済みアウトライン、前回の出力、人間からのフィードバックなど）を組み合わせて、Google Geminiモデルに最適なプロンプトを動的に生成する。
3.  **Google Gemini API呼び出し:** 構築されたプロンプトとシステム指示を用いて、Google Gemini APIを呼び出し、SOPの一部または全体を生成する。
4.  **応答の構造化とメトリクス収集:** Google Geminiモデルからの応答を構造化された形式（`LLMResult`）で返し、同時にトークン消費量や処理レイテンシなどのメトリクスを収集する。

このActivityは、SOP生成Workflowが、段階的かつ人間によるフィードバックを伴いながら、高品質なSOPを生成するための鍵となる。

#### 2.2. SOP生成Workflowにおける位置づけ
上位のTemporal Workflowからどのように呼び出され、どのようなデータを受け渡し、結果を返すのかを概説する。
本Activityは、SOP生成をオーケストレーションするTemporal Workflow（例: `SOPGenerationWorkflow`）によって呼び出される。Workflowは、SOPの生成フェーズ、関連するコンテキスト情報（ソースコード、既存のアウトライン、前フェーズの出力など）、およびユーザーからのフィードバックを引数として本Activityに渡す。本ActivityはGoogle Geminiモデルからの応答を処理し、構造化された`LLMResult`オブジェクトをWorkflowに返却する。Workflowは、この結果を受けて次のフェーズに進むか、人間によるレビューを促すなどの後続処理を行う。

### 3. エラーハンドリングとリトライポリシー
*   **対象読者:** 開発者、運用担当者
*   **所要時間の目安:** 15分
本Activityは、外部サービスであるGoogle Gemini APIに依存するため、API呼び出し時のエラーハンドリングとTemporalのRetry Policyの適切な設定が、システムの信頼性と耐障害性を確保する上で極めて重要である。

#### 3.1. ActivityにおけるRetry Policyの設定
Temporal Activityは、実行中にエラーが発生した場合に自動的に再試行する機構を持つ。`generate_sop_phase_activity`では、一時的なネットワーク障害、Google Gemini APIの一時的なサービス停止、またはレートリミット超過といった回復可能なエラーに対してリトライを適用する。

具体的な`retry_policy`は、Workflow定義内でActivity呼び出しオプションとして設定する。

```python
from datetime import timedelta
from temporalio.common import RetryPolicy
from temporalio.workflow import workflow, activity

@workflow.defn
class SOPGenerationWorkflow:
    @workflow.run
    async def run(self, initial_sop_request: dict) -> str:
        # ... 前処理 ...

        # generate_sop_phase_activity の呼び出し例
        # ActivityOptions に retry_policy を設定する
        phase_context = {"prompt": "SOPの章立てを提案してください。", "system_instruction": "あなたは熟練したドキュメントアーキテクトです。"}
        result = await activity.execute(
            "generate_sop_phase_activity", # Activity名の文字列指定
            phase_context, # Activityに渡す引数
            # ActivityOptions の設定
            schedule_to_close_timeout=timedelta(minutes=5),
            retry_policy=RetryPolicy(
                initial_interval=timedelta(seconds=1),  # 初回リトライまでの待機時間
                backoff_coefficient=2.0,                 # 指数バックオフの係数
                maximum_interval=timedelta(seconds=60),  # リトライ間隔の最大値
                maximum_attempts=5,                      # 最大リトライ回数
                # リトライすべきでないエラータイプを指定
                # Google Gemini APIの認証エラーや不正な入力はリトライしても解決しないため除外
                non_retryable_error_types=[
                    "InvalidArgumentError",   # 不正な引数（例: 無効なプロンプト）
                    "PermissionDeniedError",  # 認証/認可エラー
                    "OutOfMemoryError",       # メモリ不足（通常は環境問題だが、永続的ならリトライ無意味）
                    "BlockedPromptException", # 不適切なコンテンツによるプロンプトブロック
                ]
            )
        )
        # ... 後処理 ...
        return result["output"]
```
上記の設定により、`generate_sop_phase_activity`は、一時的なエラーに対して最大5回まで、1秒から始まり最大60秒の間隔で指数バックオフを伴うリトライを行う。一方で、`InvalidArgumentError`や`PermissionDeniedError`などの永続的なエラーに対してはリトライせず、即座にActivityの失敗としてWorkflowに通知される。

#### 3.2. Google Gemini APIエラーのハンドリング
`generate_sop_phase_activity`の実装内では、Google Gemini APIクライアントからの例外を適切に捕捉し、TemporalのRetry Policyが適用されるように`ApplicationError`などの適切な例外にラップして再スローする必要がある。これにより、API呼び出し時に発生した具体的なエラーがTemporalの例外処理機構によって認識され、定義された`retry_policy`に従って処理される。

特に、`generative_ai_protos.ServiceUnavailableError` (APIサービスの一時的な利用不可) や `generative_ai_protos.ResourceExhausted` (レートリミット超過) のような回復可能なエラーはリトライ対象とし、`InvalidArgumentError` (不正なプロンプト) や `PermissionDeniedError` (認証失敗)、`BlockedPromptException` (プロンプトブロック) のような回復不可能なエラーは`non_retryable_error_types`で指定されている場合に即座に失敗させるべきである。

```python
# activities.py (generate_sop_phase_activity 関数内)
import logging
from temporalio import activity
from temporalio.exceptions import ApplicationError
import google.api_core.exceptions as g_api_exceptions
import google.generativeai as genai
import google.generativeai.types as g_ai_types # Gemini APIのエラータイプが含まれる

# ロガーの取得 (TemporalのActivityロガーを使用)
activity_logger = activity.logger

@activity.defn
async def generate_sop_phase_activity(phase_context: dict) -> dict:
    # ... (前処理、Geminiクライアントの初期化など) ...
    # 例: genai.configure() が呼び出されている前提
    client = genai.GenerativeModel('gemini-pro') # 適切なモデルを指定

    try:
        # Google Gemini APIの呼び出し
        # 例: モデルからのテキスト生成
        response = await client.generate_content_async(
            contents=[{"role": "user", "parts": [phase_context["prompt"]]}],
            system_instruction=phase_context.get("system_instruction", "")
        )

        # 応答からテキストとトークン数を抽出
        output_text = response.text if response.text else ""
        total_tokens = response.usage_metadata.prompt_token_count + response.usage_metadata.candidates_token_count

        return {"output": output_text, "token_count": total_tokens}

    except g_api_exceptions.ServiceUnavailable as e:
        # サービス利用不可エラーは一時的である可能性が高いため、ApplicationErrorで再スローしリトライ対象とする
        activity_logger.warning(f"Google Gemini API Service Unavailable: {e}")
        raise ApplicationError(
            f"Google Gemini API Service Unavailable: {e}",
            type="ServiceUnavailableError" # Retry Policyのnon_retryable_error_typesと区別するためにエラータイプを指定
        ) from e
    except g_api_exceptions.ResourceExhausted as e: # RateLimitExceededはこれに該当することが多い
        # レートリミット超過も一時的であるため、リトライ対象とする
        activity_logger.warning(f"Google Gemini API Rate Limit Exceeded: {e}")
        raise ApplicationError(
            f"Google Gemini API Rate Limit Exceeded: {e}",
            type="RateLimitExceededError"
        ) from e
    except g_ai_types.BlockedPromptException as e:
        # 不適切なコンテンツなどによるプロンプトブロックは通常、プロンプト自体に問題があるためリトライしても無意味
        activity_logger.error(f"Google Gemini API Blocked Prompt: {e}")
        raise ApplicationError(
            f"Google Gemini API Blocked Prompt: {e}",
            type="BlockedPromptException",
            non_retryable=True # このエラーはリトライしないことを明示的に示す
        ) from e
    except g_api_exceptions.InvalidArgument as e:
        # 不正な引数エラーはプロンプトや設定の問題であり、リトライしても解決しないため非リトライ対象
        activity_logger.error(f"Google Gemini API Invalid Argument: {e}")
        raise ApplicationError(
            f"Google Gemini API Invalid Argument: {e}",
            type="InvalidArgumentError",
            non_retryable=True
        ) from e
    except Exception as e:
        # その他の予期せぬエラーは一旦汎用的なApplicationErrorとして再スローし、リトライ対象とする
        # 必要に応じて、さらに特定の例外を捕捉し、non_retryable=Trueで指定することも可能
        activity_logger.error(f"An unexpected error occurred during Google Gemini API call: {e}")
        raise ApplicationError(
            f"An unexpected error occurred: {e}",
            type="UnexpectedGeminiAPIError"
        ) from e
```

この実装により、`generate_sop_phase_activity`は回復可能なエラーに対してはTemporalのRetry Policyに従って自動的にリトライし、回復不可能なエラーに対しては直ちにWorkflowに失敗を通知する、堅牢な動作を実現する。
